# Web Presentation (바닐라 HTML 슬라이드)

번들된 검증 엔진(`assets/template.html`)을 복사해 슬라이드 콘텐츠만 채워 넣는 방식으로,
브라우저에서 바로 도는 단일 HTML 프레젠테이션을 만든다. 엔진을 매번 새로 짜지 않으므로
네비게이션·전환·전체화면 같은 까다로운 부분의 버그를 피하고 콘텐츠 구성에 집중할 수 있다.

## 기본 제공 기능 (엔진에 이미 구현됨 — 다시 만들지 말 것)

- **방향키 네비게이션**: `←`/`→`(+ PageUp/PageDown, Home/End)로 슬라이드 이동
- **F 전체화면**: `F` 키로 Fullscreen API 토글
- **슬라이드 번호 + 진행바**: 우하단 `현재 / 전체`, 상단 진행바 자동 갱신
- **Space 단계별 등장 → 다음 슬라이드**: `.fragment` 요소를 한 번에 하나씩 노출하고
  (`Shift+Space`로 되돌리기), 그 슬라이드의 단계를 모두 소진했거나 처음부터 `.fragment`가
  없으면 Space가 곧바로 다음 슬라이드로 넘어간다(마지막에 `→`를 따로 누를 필요 없음)
- **전환 애니메이션**: 슬라이드별 `data-anim="fade|slide|zoom"`
- **코드 구문 강조**: highlight.js 자동 적용 + 단계별 줄 강조
- **반응형 16:9**: 1280×720 기준 캔버스를 뷰포트에 맞춰 스케일(레터박스)
- **터치 스와이프 / URL hash deep link / 접근성(reduced-motion)**

## 작업 절차

1. **콘텐츠 파악**: 사용자가 준 내용(개요·마크다운·문서·주제)을 슬라이드 단위로 나눈다.
   분량이 모호하면 제목 1장 + 본문 여러 장 + 마무리 1장 정도로 합리적으로 구성하고,
   슬라이드 수는 내용에 맞춰 자유롭게 늘린다.
2. **템플릿 복사**: `assets/template.html`을 결과 파일(예: 사용자가 지정한 경로, 없으면
   현재 디렉토리의 `presentation.html`)로 복사한다.
3. **슬라이드 채우기**: `<!-- ▼▼▼ ... ▼▼▼ -->`와 `<!-- ▲▲▲ ... ▲▲▲ -->` 사이의
   `<section class="slide ...">` 블록을 실제 콘텐츠로 교체/추가한다. `{{TITLE}}` 류
   플레이스홀더를 모두 실제 텍스트로 채우고, `{{DECK_TITLE}}`(문서 `<title>`)도 채운다.
   남는 플레이스홀더가 없어야 한다.
4. **마감 점검**: 아래 "슬라이드 작성 규칙"을 확인하고, 코드 슬라이드가 없으면 `<head>`의
   highlight.js `<link>`/`<script>` 3줄을 삭제(불필요한 외부 의존 제거)한다.
5. **안내**: 결과 파일 경로와 조작법(`←/→` 이동, `F` 전체화면, `Space` 단계 등장)을 알려준다.
   브라우저로 바로 열 수 있고(`file://`), 코드 강조까지 쓰면 인터넷 연결이 필요함을 덧붙인다.

## 슬라이드 작성 규칙

각 슬라이드는 다음 형태다. `data-anim`으로 전환 효과를, `bg-*` 클래스로 배경을 고른다:

```html
<section class="slide bg-purple" data-anim="slide" aria-label="슬라이드 N">
  <div>
    <h2>섹션 제목</h2>
    <ul>
      <li class="fragment" style="--i:0">첫 번째 포인트</li>
      <li class="fragment" style="--i:1">두 번째 포인트</li>
    </ul>
  </div>
</section>
```

- **콘텐츠는 항상 하나의 `<div>`로 감싼다** — 슬라이드는 grid 중앙정렬이라 단일 자식이어야 깔끔히 가운데 온다.
- **전환 효과** `data-anim`: `fade`(기본·차분), `slide`(좌우 이동), `zoom`(확대). 한 덱 안에서 섞어 써도 된다.
- **배경 클래스**: `bg-blue` `bg-purple` `bg-green` `bg-red` `bg-teal`. 생략하면 기본 어두운 배경.
- **단계별 등장**: 한 번에 하나씩 보여줄 요소에 `class="fragment"`를 붙이고 `style="--i:0"`,`--i:1`…로
  등장 지연(stagger)을 준다. 첫 화면부터 보여줄 내용엔 붙이지 않는다.
- **첫 슬라이드**에는 `active` 클래스를 유지한다(처음 표시될 슬라이드). 나머지엔 붙이지 않는다.
- **제목 슬라이드**는 `<h1>`, **본문 슬라이드**는 `<h2>` + `<ul>`/`<p>`, **강조 숫자**는 `class="big"`.

### 코드 슬라이드

코드를 보여줄 땐 `<pre class="code-block">` + `<code class="language-자바스크립트등">`을 쓴다.
구문 색상은 highlight.js가 자동 처리한다(언어 클래스 지정 권장). 코드는 반드시 그대로 텍스트로 넣어
HTML 특수문자(`<`, `>`, `&`)가 깨지지 않게 한다.

`data-line-numbers`로 Space를 누를 때마다 강조할 줄을 단계별로 지정할 수 있다(선택):

```html
<pre class="code-block" data-line-numbers="1|2,3|5"><code class="language-python">def add(a, b):
    total = a + b
    return total

print(add(1, 2))</code></pre>
```

`"1|2,3|5"` = 1단계는 1번 줄, 2단계는 2·3번 줄, 3단계는 5번 줄을 강조(범위는 `2-4` 형식). 단계별
강조가 필요 없으면 `data-line-numbers` 속성을 빼면 된다.

## 엔진을 건드려야 할 때

기본값(전환 속도, 색, 폰트 크기, 키 매핑 등)은 `<style>`과 키보드 `switch` 문에서 조정한다.
구조(슬라이드 겹침 + `.active` 토글, `goTo`/`applyStep` 단일 상태 흐름)는 검증된 패턴이므로
특별한 이유 없이 바꾸지 말 것 — 입력 잠금(`isAnimating`)·`transitionend` 폴백 등 미묘한 버그를
이미 처리해 두었다.

설계 배경과 브라우저 호환성 근거가 필요하면 `references/architecture.md`를 참고한다.
